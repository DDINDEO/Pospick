# 🎨 ComicWorld POS

> 코믹월드 등 동인 행사 부스 운영 작가를 위한 모바일 웹 POS 시스템

현금과 계좌이체 중심의 동인 행사 환경에 특화된 POS로,  
기존 포스 앱이 해결하지 못했던 **계좌이체 플로우**, **행사 단위 운영**, **굿즈 이미지 UI**를 제공합니다.

---

## 📌 프로젝트 배경

코믹월드를 비롯한 동인 행사에서 부스를 운영하는 작가들은 대부분 현금과 계좌이체로 거래합니다.  
그러나 기존 모바일 POS 앱(토스 포스, IBK BOX POS 등)은 **카드 단말기·사업자등록·월 구독료**를 전제로 설계되어 있어 개인 작가에게는 맞지 않습니다.

**이 프로젝트는 다음 문제를 해결합니다.**

- 암산 실수와 거스름돈 계산 오류
- 계좌이체 입금 확인 대기 중 다음 손님을 못 보는 문제
- 행사 종료 후 수기로 매출을 정리하는 번거로움
- 재고가 얼마 남았는지 파악하지 못하는 상황

## ✨ 주요 기능

### 결제 처리
| 결제 방식 | 기능 |
|----------|------|
| 현금 | 상품 탭 → 합산 → 거스름돈 자동 계산 |
| 계좌이체 | QR 즉시 표시 → 입금 대기 / 확인 / 취소 3가지 상태 관리 |
| 복합 결제 | 현금 먼저 입력 → 잔액 자동 계산 → 계좌이체 QR 표시 |

### 상품 관리
- 이미지 + 이름 + 가격 + 재고 수량 등록
- 이미지 기반 그리드 UI (손님이 "저거요" 하면 바로 탭)
- 재고 매진 임박 경고 / 행사 중 재고 수동 추가

### 행사 단위 운영
- 행사 시작 / 종료 모드
- 현금 준비금 입력 → 거스름돈 부족 시점 사전 경고
- 회차별 데이터 자동 분리 저장

### 매출 리포트 (행사 종료 후 자동 정산)
- 총 매출 (현금 / 계좌이체 / 복합 분리)
- 상품별 판매량 및 완판 상품 표시
- 시간대별 판매 흐름
- 거스름돈 지출 합계

### 행사 주최측 기능
- 행사 생성 및 데이터 수집 옵션 설정
- 참가 신청 승인 / 거절
- 동의된 옵션 범위 내 전체 부스 현황 대시보드

## 🏗 시스템 구조

```
주최측 (Organizer)          판매자 (Seller)
       │                          │
       └──────────┬───────────────┘
                  │
          React + TypeScript
          (Firebase SDK)
                  │
    ┌─────────────┼─────────────┐
    │             │             │
Firebase Auth  Firestore  Security Rules
(로그인/권한)  (실시간 DB)  (접근 제어)
                  │
          Firebase Hosting
          (무료 · URL 접속만으로 실행)
```

### Firestore 데이터 구조

```
users/          → role: "organizer" | "seller"
events/         → 행사 정보 + 데이터 수집 옵션
applications/   → 참가 신청 + 동의 내역 (timestamp 영구 기록)
booths/         → 상품 / 매출 / 재고
```

```
[주최측]                        [판매자]

1. 회원가입 (organizer)
2. 행사 생성
   - 수집 옵션 설정
     ☑ 총 매출 공개
     ☑ 상품별 판매량 공개
     ☐ 재고 현황 공개
3. 참가 신청 오픈
                                4. 회원가입 (seller)
                                5. 행사 검색 / 링크 접속
                                6. 수집 옵션 확인 후 동의
                                7. 참가 신청
8. 신청 목록 확인
9. 승인 → POS 활성화
                                10. 상품 등록
                                11. 행사 시작
                                12. 결제 처리 (현금 / 계좌이체 / 복합)
10. 대시보드에서               13. 행사 종료
    전체 부스 현황 조회             → 자동 정산 리포트 확인
```

## ⚙️ 기술 스택

| 분류 | 기술 |
|------|------|
| Frontend | React + TypeScript |
| Styling | Tailwind CSS |
| 인증 | Firebase Auth |
| 데이터베이스 | Firebase Firestore |
| 호스팅 | Firebase Hosting |
| 오프라인 | PWA (Service Worker + 오프라인 캐싱) |

> **백엔드 서버를 직접 운영하지 않습니다.**  
> Firebase가 인증·DB·권한 관리를 모두 담당합니다. (BaaS 구조)

## 📐 결제 시나리오

### 시나리오 A — 현금 결제
1. 손님이 굿즈 선택 → 작가가 상품 탭 → 금액 자동 합산
2. 받은 금액 입력 → 거스름돈 자동 표시
3. 거스름돈 전달 → 거래 완료 / 매출 기록

### 시나리오 B — 계좌이체
1. 손님이 계좌이체 요청 → 계좌 QR 화면 즉시 표시
2. 손님 이체 → 입금 대기 상태 유지 (다음 손님 응대 가능)
3. 입금 확인 버튼 탭 → 완료

### 시나리오 C — 복합 결제
1. 현금 받은 금액 입력 → 잔액 자동 계산
2. 잔액에 대한 계좌이체 QR 표시 → 이체 확인 → 완료

---

## 🔒 데이터 공개 원칙

- 주최측은 판매자가 **사전에 동의한 옵션 범위 내** 데이터만 열람 가능
- 동의 내역은 `agreedAt` timestamp와 함께 Firestore에 영구 기록
- 판매자는 본인 데이터 전체 열람 가능

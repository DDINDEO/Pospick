# 🎨 POSPICK
> 행사 부스 운영 작가를 위한 모바일 웹 POS 시스템

현금과 계좌이체 중심의 동인 행사 환경에 특화된 POS로,  
기존 포스 앱이 해결하지 못했던 **계좌이체 플로우**, **행사 단위 운영**, **굿즈 이미지 UI**를 제공합니다.

---

## 📌 프로젝트 배경

코믹월드를 비롯한 동인 행사에서 부스를 운영하는 작가들은 대부분 현금과 계좌이체로 거래합니다.  
그러나 기존 모바일 POS 앱(토스 포스, IBK BOX POS 등)은 **카드 단말기·사업자등록·월 구독료**를 전제로 설계되어 있어 개인 작가에게는 맞지 않습니다.

**이 프로젝트는 다음 문제를 해결합니다.**

- 암산 실수와 거스름돈 계산 오류
- 행사 종료 후 수기로 매출을 정리하는 번거로움
- 재고가 얼마 남았는지 파악하지 못하는 상황
- 행사 관리자가 어떤 물품이 잘 팔렸는지 파악하지 못하는 경우

---

## ✨ 주요 기능

### 상품 관리
- 이미지 + 이름 + 가격 + 재고 수량 등록
- 이미지 기반 그리드 UI (손님이 "저거요" 하면 바로 탭)
- 재고 매진 임박 경고 / 행사 중 재고 수동 추가
- 현금계산 상품 탭 → 합산 → 거스름돈 자동 계산
- API를 이용하여 상품 자동분류

### 행사 단위 운영
- 행사 시작 / 종료 모드
- 회차별 데이터 자동 분리 저장

### 매출 리포트 (행사 종료 후 자동 정산)
- 총 매출
- 상품별 판매량 및 완판 상품 표시
- 시간대별 판매 흐름
- 거스름돈 지출 합계

### 행사 주최측 기능
- 행사 생성 및 데이터 수집 옵션 설정
- 참가 신청 승인 / 거절
- 동의된 옵션 범위 내 전체 부스 현황 대시보드

## 🛠 기술 스택 (Tech Stack)
* **Backend**: Java, Spring Boot, Spring Data JPA, Spring Security
* **Frontend**: React, TypeScript, Tailwind CSS
* **Database**: MySQL
* **AI API**: Google Gemini API
* **Storage**: Local File System (추후 S3 확장 가능 구조) (추후 무엇을 사용할지 결정)

## 📅 개발 로드맵 (12 Weeks)

### 🏗️ Phase 1: 환경 설정 및 핵심 도메인 설계 (Week 1 - 3)
- [ ] **1주차: 프로젝트 뼈대 및 DB 설계**
  - [ ] 핵심 엔티티 정의 및 MySQL ERD 설계 (User, Event, Product 관계)
  - [ ] Spring Boot & React 기본 프로젝트 생성 및 DB 연결 확인
- [ ] **2주차: 회원 및 행사 기본 CRUD**
  - [ ] 회원가입 / 로그인 기본 로직 (API 동작 중심)
  - [ ] 주최측(Organizer): 행사 생성 및 조회 기능 구현
- [ ] **3주차: 인증 시스템 도입 (Spring Security)**
  - [ ] JWT를 이용한 로그인 기능 및 토큰 관리 구현
  - [ ] Organizer / Seller 권한 분리 및 접근 제어 적용

### 🤝 Phase 2: 참가 신청 및 서비스 흐름 연결 (Week 4 - 5)
- [ ] **4주차: 행사 참가 프로세스**
  - [ ] 판매자(Seller): 행사 검색 및 참가 신청 기능

### 🤖 Phase 3: AI 상품 관리 및 스토리지 (Week 6 - 8)

## 📐 결제 시나리오

### 시나리오 A — 현금 결제
1. 손님이 굿즈 선택 → 작가가 상품 탭 → 금액 자동 합산
2. 받은 금액 입력 → 거스름돈 자동 표시
3. 거스름돈 전달 → 거래 완료 / 매출 기록

### 시나리오 B — 계좌이체
1. 손님이 굿즈 선택 → 작가가 상품 탭 → 금액 자동 합산
2. 손님 이체 → 거래 완료 / 매출 기록
3. 완료

## 🔒 데이터 공개 및 보안 원칙 (Data Privacy Policy)

본 프로젝트는 주최측과 판매자 간의 신뢰 가능한 데이터 공유를 위해 다음 원칙을 준수합니다.

1. **최소 권한 기반 데이터 열람**
   - 주최측(Organizer)은 판매자가 **사전에 동의한 수집 옵션 범위 내** 데이터(예: 총 매출, 카테고리별 판매량 등)만 열람할 수 있습니다.
   - 판매자가 동의하지 않은 세부 데이터(개별 거래 내역 등)는 주최측 대시보드에서 마스킹 처리되거나 조회되지 않습니다.

2. **동의 내역 기록 및 무결성 보장**
   - 모든 데이터 수집 동의 내역은 `agreedAt` 타임스탬프와 함께 DB에 영구 기록됩니다.
   - 동의 여부는 행사 참가 신청 시점에 확정되며, 이후 데이터 열람 권한의 근거가 됩니다.

3. **판매자 데이터 주권 보장**
   - 판매자(Seller)는 본인이 운영한 부스의 전체 데이터(이미지, 상세 거래 내역, 재고 현황 등)를 제한 없이 열람하고 관리할 권리를 가집니다.
